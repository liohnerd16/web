# GUIDE TO EDITING CODE & MANAGING CATEGORIES / HƯỚNG DẪN SỬA CODE & QUẢN LÝ DANH MỤC

Primary file for editing: `server/database/db.js`
File chỉnh sửa chính: `server/database/db.js`

---

## 1. Data Management / Quản lý Dữ liệu

### Local Development (SQLite)
- Database được lưu tại `server/database/data.db`
- `initDb()` tự động tạo bảng và seed data sample (13 videos + 6 updates)

### Production on Vercel
- **Có Turso**: Dữ liệu persistent trên cloud, cần `TURSO_DATABASE_URL` + `TURSO_AUTH_TOKEN`
- **Không Turso**: SQLite fallback tại `/tmp/data.db` (detect `process.env.VERCEL`). Seed data chạy mỗi cold start. Dữ liệu người dùng KHÔNG persistent nếu không có Turso.

### Sync code changes to live site:
- Commit & push to GitHub → Vercel auto-deploys
- Hoặc: `vercel --prod` (nếu dùng Vercel CLI)

---

## 2. Managing Categories
Categories are generated **automatically** from your video entries. 

### To Add or Rename:
1. Update the category in `db.js` (samples array, index 3-4 for EN/VI).
2. Add I18N key in `public/script.js` if needed.
3. Deploy to Vercel.

---

## 3. Multilingual Text (I18N)
UI text (buttons, menus, toasts) is managed in: `public/script.js`.

1. Find the `const I18N = { ... }` variable.
2. Edit English (`en`) or Vietnamese (`vi`) keys.
3. To auto-sync translations: `npm run translate`.

---

## 4. Dashboard & Project Editing
The dashboard includes advanced tools for content management:

- **Bilingual Edit**: The edit modal allows updating both EN and VI versions of titles, descriptions, and topics simultaneously.
- **Live LaTeX Preview**: Click the "Preview" button in the edit modal to see a live rendering of the scientific explanation (including MathJax and diagrams) before saving.
- **Localized Table**: The project management table is fully bilingual and switches based on the global language toggle.

---

## 5. Maintenance Workflow (IMPORTANT)
Before pushing any major code changes:
1. **Build**: `npm run build` (minify JS/CSS + optimize SVGs).
2. **Check syntax**: `node --check server/*.js` (và các file đã sửa).
3. **Sync Manifest**: `npm run sync-manifest` (tạo "What's New" entries).
4. **Deploy**: Commit & push to GitHub (Vercel auto-deploys) hoặc `vercel --prod`.
5. **Lưu ý**: `check-images` và `check-style` là legacy — KHÔNG chạy.

---

## 6. Security (Secrets)
Never hardcode API Keys. Manage them via:
- **Local**: `.env` file.
- **Production**: Vercel Dashboard → Project Settings → Environment Variables.

## 7. Vercel-specific Notes
- **Proxy route**: `/api/proxy-image` chạy TRƯỚC DB middleware — logo/favicon độc lập với DB.
- **Logo local**: `public/assets/images/ui/gnz-logo.png` — không phụ thuộc Google Drive.
- **Build output**: HTML references `script.min.js` và `styles.min.css`. Chạy `npm run build` trước deploy.
